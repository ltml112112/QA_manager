SECTION 01
왜 만드는가 — 현재 업무의 단절
지금 품질 데이터·평가 결과·Lot 이력이 각자 다른 도구에 흩어져 있음. 같은 Lot의 정보가 5~6개 도구를 거쳐 통합되어야만 출하 의사결정 가능.
영역현재 도구문제점
소자평가 결과이메일 + 엑셀 첨부누락·재집계 빈번, 검색 불가
Lot 이관 일정카카오톡 + 메일 표부서 간 요청 추적 어려움
Lot 계보 추적사내 ERP 의존TREND 분석과 연계 불가
SQC TREND엑셀 별도 관리이상치 발생 시 원인 역추적 안 됨
COA 발행수기 작성 + PDF 변환발행 시간 ↑, 휴먼 에러
사전심사 자료시트 매크로 (LGD/SDC)양식 변경 시 수동 대응
결과 — 통합 허브가 필요함. 데이터를 한 곳에 모으고, 한 흐름으로 흐르게 만든다.

SECTION 05
핵심 통합 사례 — "이미 작동하고 있는 통합"
각 앱이 따로 떨어져 있지 않고, 데이터가 자동으로 흐르고 있는 두 가지 사례.
Case A · 01 ↔ 06 평가 흐름 통합
  1. 정제팀이 Lot 이관 일정 등록 (06)
  2. 평가 담당자가 CSV 분석 (01, embed)
  3. "이 Lot에 결과 저장" 클릭
  4. Firebase에 결과 저장 → 06 카드에 배지 자동 표시
  5. 10번 대시보드에 KPI 자동 반영
→ 이미 데이터가 한 흐름으로 흐르고 있음
Case B · 05 계보 + TREND 통합
  1. 흐름도 Excel 업로드 → 모든 Lot 계보 자동 구성
  2. 품질 Excel 업로드 → 같은 Batch No. 매칭
  3. SQC 차트에서 이탈 포인트 클릭
  4. Batch Deep-Dive → 전 공정 측정값 즉시 확인
  5. 계보 클립보드 복사 → Excel rowspan 그대로 붙여넣기
→ 이상치 발생 → 원인 역추적이 1분 안에 가능
Case A 화면
OLED 결과 → Lot 카드 배지
분석기(01)에서 "결과 저장" 클릭 → postMessage → 06 카드에 LT% 배지 자동 표시.
→ mockups/06_oled_badge_zoom.html
 Case B 화면
Batch Deep-Dive (이탈 분석)
SPC 이탈 → Deep-Dive → 전 공정 측정값 + 규격 비교 + 이탈 원인 분석 패널.
→ mockups/05_deepdive.html
SECTION 06
Phase 2 — COA 자동 발행 (다음 6개월 목표)
현재 흩어져 있는 데이터를 하나의 발행 엔진으로 묶어, 출하 COA의 단일 진실 공급원(Single Source of Truth) 구축.

● COA Engine 구조

입력 데이터:
├─ 소자평가 결과 (oled_results)
├─ 소재 Spec & CTQ (App 12, NEW)
├─ Batch별 측정값 (App 05)
├─ 외부 코드 매핑 (App 09, NEW)
└─ 원자재 IQC 이력 (App 13, NEW)

         ↓ COA Engine ↓

출력:
├─ COA 양산용 (App 08) ── 고객 발송
└─ COA 개발용 (App 07) ── 내부 검토

● 선결 조건 — 데이터 표준화

  • App 12 — 제품 Spec & CTQ/CTP소재별 규격·핵심 품질/공정 파라미터 · COA 항목·SPC 관리 한계의 단일 정의
  • App 13 — 원자재 IQC입고 검사 결과를 Lot 계보 시작점으로 등록
  • App 09 — 외부코드 관리고객사별 외부 코드 ↔ 내부 코드 매핑 · COA 발행 시 고객 코드 자동 치환
Phase 2 완료 시점 = 첫 자동 COA 발행 가능 시점

SECTION 08
Phase 3 — 실시간 Trend & 역추적 허브
데이터가 충분히 쌓이면 사후 대응 → 사전 예방으로 전환. 컴플레인 역추적 + 자동 알림 + 표준 정의 자동 흐름.

● 컴플레인 → 원인 역추적

  • App 11 — 컴플레인 관리고객 컴플레인 등록 시 Lot 번호로 시작
  • App 05 계보를 따라 합성·정제 단계까지 자동 역추적
  • 이탈 측정값 / 변경된 공정 단계 자동 표시

● 실시간 SPC 알림

  • App 10 대시보드에서 관리도 이탈 발생 시
  • Slack / 메일 알림 → 사후 대응 → 사전 예방으로 전환

● 사전심사 자동화 확장

  • App 04 — SDC 사전심사LGD(02)와 동일 패턴으로 GAS 기반 발행 · 양식·증빙 자료 통합

● 데이터 표준화 효과

  • CTQ/CTP가 정의되면 자동 연쇄:
  • App 05 SQC 관리한계 자동 적용
  • App 07/08 COA 항목 자동 구성
  • App 10 KPI 정의 자동 반영
  • App 13 IQC 판정 기준 자동 적용
→ 한 번 정의된 규격이 모든 모듈에 흐른다

기술 스택 — 단순함이 곧 지속가능성
100% 클라이언트 사이드 · 운영 비용 거의 0원 · 모든 코드가 Git 관리.

● 인프라

  • 호스팅: Cloudflare Pages (정적, 무상)
  • 인증: Firebase Auth (Email/Pw, @ltml.co.kr 도메인)
  • DB: Firebase Realtime Database
  • 자동화: Google Apps Script (LGD/SDC)
  • 연산: 100% 클라이언트 사이드 (개인정보 외부 미전송)

● 보안

  • 도메인 화이트리스트 (@ltml.co.kr 만 가능)
  • 역할 분리 (admin / user)
  • 직접 URL 접근 차단 (auth_guard.js 이중 방어)
  • 관리자만 계정 생성 가능 (회원가입 비활성화)

● 운영 비용 — 거의 0원

  • Cloudflare Pages: 무료 플랜으로 충분
  • Firebase: Spark 플랜(무료) 한도 내
  • GAS: Google Workspace에 포함

● 유지보수성

  • 새 도구 추가: apps/ 폴더 하나 + main.js 한 줄
  • 테마 변경: global_style.css 변수만 수정
  • 모든 코드가 Git 관리: claude/... 브랜치 → main PR
  • CLAUDE.md — 코드베이스 자체 문서화
