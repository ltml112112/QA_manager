SECTION 01
왜 만드는가 — 현재 업무의 데이터 파편화
지금 품질 데이터·평가 결과·Lot 이력이 각자 다른 도구에 흩어져 있음. 같은 Lot의 정보가 여러 시스템을 거쳐 통합되어야만 출하 의사결정이 가능한 구조.
영역현재 도구문제점
소자평가 결과이메일 + 엑셀 첨부누락·재집계 빈번, 검색 불가
Lot 이관 일정카카오톡 + 메일 표부서 간 요청 추적 어려움
Lot 계보 추적사내 ERP + 합성생산팀 수기 관리TREND 분석 연계 불가
SQC TREND엑셀 별도 관리이상치 발생 시 원인 역추적 안 됨
COA 발행엑셀 VBA고객사·제품별 양식 변경 시 수동 대응 한계
사전심사 자료 (LGD/SDC)엑셀 VBA양식 변경 시 매크로 매번 수정
결론 — 데이터를 한 곳에 모으는 중심 축이 필요함. 입력부터 출력까지 한 흐름으로 연결되어야 함.

SECTION 06
COA 자동 발행 — 유연한 양식 대응
현재 다양한 고객사가 다양한 form 의 COA sheet를 요청하고 있음. 기존 정형화된 엑셀 VBA로는 대응에 한계. 입력 데이터 항목도 점점 많아지고 있음.

● 기존 → 현재의 변화

  • 예전: HPLC 순도 1개 항목만 제출정형화된 엑셀 양식으로 충분히 대응 가능
  • 지금: 항목이 폭증 — 다양한 측정값이 동시에 요구됨
최근 요청되는 입력 항목 예시:
HPLC 순도
지정 imp peak
DSC
소자평가 (IVL)
소자평가 (LT)
LT chart
입도 분석
RoHS
Metal·할로겐

● 해결 방향

  • 데이터를 하나로 묶음원재료 IQC부터 Lot 추적·대시보드까지 한 흐름
  • 유연한 양식 엔진고객사·제품별로 다른 항목 조합을 동적으로 구성
  • 자동 채움한 번 측정·등록된 값은 모든 양식에 자동 반영
  • Spec 기반 자동 판정CTQ/CTP 정의에 따라 OK/NG 자동 표시
핵심 — 원재료 IQC부터 COA 발행, 그 이후 Lot 추적·대시보드까지 전 과정을 한 줄로 묶는 것이 이 시스템의 가장 중요한 목표.
SECTION 07
데이터 통합 시나리오 — Lot ANA0101 출하까지
완성 시점에서 입고부터 COA 발행, 출하 후 Lot 추적·대시보드 반영까지 전 흐름이 한 시스템에 기록되는 모습.
[ 1 ]  원재료 입고
       → App 13 (IQC)            — Lot 시작점 등록 (입고 검사값)

[ 2 ]  합성 / 정제 진행
       → App 15 (P/N Flow)       — 공정 단계·조건 기록
       → App 03 (HPLC/DSC)       — 분석 데이터 등록

[ 3 ]  정제 완료 → 소자평가 의뢰
       → App 06 (Lot 일정)       — 이관 일정 등록
       → App 01 (OLED 분석)      — IVL · LT 결과 자동 저장

[ 4 ]  품질 데이터 종합
       → App 05 (Lot 추적 + SQC) — 계보·TREND 검증
       → App 12 (Spec/CTQ)       — 규격 자동 비교 → OK/NG 판정

[ 5 ]  출하 COA 발행
       → App 08 (COA 양산)       — 고객사 양식에 맞춘 PDF 자동 발행
       → 한 번 입력된 데이터가 양식에 자동 채워짐

[ 6 ]  출하 이후
       → App 10 (대시보드)       — KPI·TREND 자동 반영
       → App 05 (Lot 추적)       — 컴플레인 발생 시 즉시 역추적
목표 — 입고부터 출하 COA 발행, 그 이후 Lot 추적까지 모든 흔적이 한 시스템에 남는다.
SECTION 08
Trend & 보고 시스템 — 데이터를 잘 보는 시스템
데이터가 쌓이면 끝이 아님. 잘 보고, 잘 활용해야 함. TREND 시각화와 정기 보고를 위한 시스템 구축.

● TREND 시각화 강화

  • SQC 차트 고도화관리 한계·이탈·추세를 한눈에
  • Batch Deep-Dive이탈 포인트 → 원인 후보 즉시 확인
  • 계보 + TREND 동시 조회이상치 발생 시 어느 단계에서 시작됐는지 추적

● 정기 보고 자동화

  • 주간 / 월간 / 연간 보고대시보드 데이터를 그대로 보고 자료로
  • 리포트 자동 생성 → 수기 작성 시간 제거

● 유연한 시스템 — 표준화의 본질

  • 재료별 CTQ/CTP가 다 다름
  • COA 항목이 제품별·고객사별로 점점 다양해짐
  • 이런 변화에 유연하게 대응하기 편한 시스템이 필요
엄격한 표준화가 아니라, 다양성에 유연하게 대응하는 구조 — 이게 핵심 설계 원칙.
SECTION 09

각 단계는 이전 단계의 데이터 위에 쌓임. 하나하나가 독립적으로 가치를 내면서, 다음 단계의 기반이 됨.
진행 중
핵심 모듈 구축
소자평가 일정·OLED 분석은 운영 중. Lot 추적·SQC, 품질 대시보드, P/N Flow를 구축 중. 데이터 통합의 뼈대를 다지는 단계.
소자평가 Lot 일정 OLED IVL & LT Lot 추적 & SQC 품질 대시보드 P/N Flow HPLC/DSC Report
이어서

원재료 IQC Spec & CTQ/CTP COA 양산용 COA 개발용 SDC 사전심사

SECTION 10
기대 효과 — 사람의 역할 재정의
단순한 시간 단축이 아님. 한 사람이 도맡아 하던 일을 시스템이 대신함으로써, 그 사람의 시간을 더 가치 있는 일에 쓸 수 있게 만드는 것이 본질.
★
"COA 발행 담당자"라는 역할이 사라짐
파편화된 데이터를 한 사람이 모아서 양식에 채워 넣는 일이 시스템으로 대체됨. 그 인력이 더 가치 있는 분석·기획 업무로 이동 가능.
⊘
휴먼 에러 0
사람이 데이터를 타이핑하는 것, imp peak sorting, 소자평가 결과 수동 계산 — 전체 시스템을 전산화하면서 휴먼 에러가 원천 제거됨.
✓
데이터 신뢰성 향상
한 번 입력된 데이터가 모든 출력에 동일하게 흐름. 같은 Lot에 대해 시스템마다 값이 다른 일이 없어짐.
⤴
추적성 (Traceability)
출하 후 어떤 컴플레인이 와도 원재료 IQC까지 역추적 가능. Lot 계보 + TREND가 한 시스템 안에 있음.
⚡
의사결정 속도 향상
대시보드·SQC TREND를 실시간으로 보면서 의사결정. 자료 취합 → 분석 → 보고의 시간차가 사라짐.
⏱
시간 단축 (부수 효과)
COA 발행, Lot 이력 추적, 컴플레인 원인 추적, 사전심사 자료 작성 — 모든 영역에서 획기적인 시간 단축 동반.
핵심 — 시간 단축은 결과일 뿐, 본질은 "사람이 데이터를 옮기는 일"을 시스템에게 맡기는 것. 그래서 사람은 데이터를 보고 판단하는 일에 집중할 수 있게 됨.
