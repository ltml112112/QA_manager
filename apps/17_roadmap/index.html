SECTION 01
추진 배경
현재 품질 데이터, 평가 결과, Lot 이력은 이메일·엑셀·ERP 등 여러 도구에 나뉘어 관리되고 있습니다. 같은 Lot에 대한 정보를 모으려면 매번 여러 시스템을 거쳐야 출하 판단이 가능한 구조입니다.
영역현재 도구문제점
소자평가 결과이메일 + 엑셀 첨부누락이 잦고 검색이 어려움
Lot 이관 일정카카오톡 + 메일 표부서 간 요청 추적이 어려움
Lot 계보 추적사내 ERP + 합성생산팀 수기 관리TREND 분석과의 연계가 끊겨 있음
SQC TREND엑셀 별도 관리이상치 발생 시 원인 역추적이 어려움
COA 발행엑셀 VBA고객사·제품별 양식 변경 대응에 한계
사전심사 자료(LGD/SDC)엑셀 VBA양식 변경마다 매크로를 재작성해야 함
각 데이터를 한 곳에 모으고, 입력부터 출력까지 하나의 흐름으로 연결할 수 있는 시스템이 필요하다고 판단됩니다.

SECTION 06
COA 자동 발행 — 다양한 양식 대응
최근 고객사별로 요청되는 COA 양식이 다양해지고, 포함되어야 할 측정 항목 또한 빠르게 늘어나고 있습니다. 기존 엑셀 VBA 방식은 양식이 바뀔 때마다 매크로를 다시 손봐야 해 대응이 점점 어려워지는 상황입니다.

요구 사항의 변화

  • 예전 · HPLC 순도 1개 항목만 제출하면 충분했던 시기정형화된 엑셀 양식으로도 대응이 가능했습니다
  • 현재 · 측정 항목이 다양해지고 동시에 요구되는 추세입니다
최근 요청 항목 예시
HPLC 순도
지정 imp peak
DSC
소자평가 (IVL)
소자평가 (LT)
LT chart
입도 분석
RoHS
Metal · 할로겐

해결 방향

  • 데이터 통합원재료 IQC부터 Lot 추적·대시보드까지 단일 흐름으로 연결합니다
  • 양식 엔진고객사·제품별 항목 조합을 동적으로 구성할 수 있는 구조로 설계합니다
  • 자동 채움한 번 측정·등록된 값은 모든 양식에 자동 반영됩니다
  • 자동 판정CTQ/CTP 정의에 따라 OK/NG가 자동으로 표시됩니다
원재료 입고부터 COA 발행, 그리고 출하 후 Lot 추적·대시보드까지 전 과정을 하나의 데이터 흐름으로 묶는 것이 본 시스템의 가장 중요한 목표입니다.
SECTION 07
통합 시나리오 — Lot ANA0101 출하까지
구축이 완료되었을 때 하나의 Lot이 입고부터 출하·사후 추적까지 어떻게 시스템에 기록되는지 보여주는 시나리오입니다. 
[ 1 ]  원재료 입고
       → App 13 (IQC)            — Lot 시작점 등록 (입고 검사값)

[ 2 ]  합성 / 정제 진행
       → App 15 (P/N Flow)       — 공정 단계 · 조건 기록
       → App 03 (HPLC/DSC)       — 분석 데이터 등록

[ 3 ]  정제 완료 → 소자평가 의뢰
       → App 06 (Lot 일정)       — 이관 일정 등록
       → App 01 (OLED 분석)      — IVL · LT 결과 자동 저장

[ 4 ]  품질 데이터 종합
       → App 05 (Lot 추적 + SQC) — 계보 · TREND 검증
       → App 12 (Spec/CTQ)       — 규격 자동 비교 → OK/NG 판정

[ 5 ]  출하 COA 발행
       → App 08 (COA 양산)       — 고객사 양식에 맞춘 PDF 자동 발행
       → 한 번 입력된 데이터가 양식에 자동으로 채워집니다

[ 6 ]  출하 이후
       → App 10 (대시보드)       — KPI · TREND 자동 반영
       → App 05 (Lot 추적)       — 컴플레인 발생 시 즉시 역추적
입고부터 출하 후 사후 추적까지 모든 이력이 단일 시스템에 누적됩니다.
SECTION 08
Trend · 보고 — 데이터를 활용하는 단계
데이터를 쌓는 것 자체로는 의미가 크지 않습니다. 추세를 시각화하고 정기 보고로 연결되어야 실제 의사결정에 활용될 수 있습니다.

TREND 시각화

  • SQC 차트 고도화관리 한계, 이탈, 추세를 한 화면에서 확인합니다
  • Batch Deep-Dive이탈 포인트에서 원인 후보를 즉시 확인할 수 있습니다
  • 계보 + TREND 동시 조회이상치 발생 시 어느 단계에서 시작되었는지 추적합니다

정기 보고

  • 주간 / 월간 / 연간 보고대시보드 데이터를 그대로 보고 자료에 활용합니다
  • 리포트 자동 생성으로 수기 작성 시간을 줄입니다

유연한 시스템 — 설계 원칙

  • 재료별 CTQ/CTP는 모두 다릅니다
  • COA 항목은 제품별·고객사별로 점차 다양해지고 있습니다
  • 이러한 변화에 부담 없이 대응할 수 있는 구조가 필요합니다
재료마다 관리 항목이 다르고 고객사마다 요구 양식이 다른 현실을 전제로, 엄격한 표준화보다 변화에 유연하게 대응할 수 있는 구조를 우선했습니다.

SECTION 10
기대 효과
단순히 업무 시간을 줄이는 것이 목적은 아닙니다. 한 명이 도맡아 처리하던 단순 취합 업무를 시스템에 맡기고, 사람의 시간을 분석과 판단처럼 더 가치 있는 일에 쓸 수 있도록 만드는 것이 본 시스템의 지향점입니다.
★
COA 발행 업무의 자동화
여러 시스템을 오가며 데이터를 모아 양식에 채우는 작업이 시스템으로 대체됩니다. 해당 인력은 분석·기획 업무로 재배치할 수 있습니다.
⊘
휴먼 에러 감소
수기 타이핑, imp peak 정리, 소자평가 결과 수동 계산 등 사람이 개입하던 단계를 전산화하여 입력 오류를 원천적으로 줄입니다.
✓
데이터 신뢰성 향상
한 번 입력된 데이터가 모든 출력에 동일하게 반영됩니다. 같은 Lot에 대해 시스템마다 값이 다른 상황은 발생하지 않습니다.
⤴
추적성 (Traceability)
출하 후 컴플레인이 접수되었을 때 원재료 IQC 단계까지 역추적이 가능합니다. Lot 계보와 TREND가 동일 시스템에 보관됩니다.
⚡
의사결정 속도 향상
대시보드와 SQC TREND를 실시간으로 확인하면서 의사결정이 가능합니다. 자료 취합부터 보고까지 발생하던 시간차가 줄어듭니다.
⏱
업무 시간 단축
COA 발행, Lot 이력 추적, 컴플레인 원인 추적, 사전심사 자료 작성 등 모든 영역에서 작업 시간이 단축됩니다.
시간 단축은 결과로 따라오는 효과입니다. 본 시스템이 지향하는 것은 사람이 데이터를 옮기는 작업에서 벗어나, 데이터를 보고 판단하는 일에 집중할 수 있는 환경을 만드는 일입니다.
