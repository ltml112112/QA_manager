# QA Manager Portal 개선 제안 (2026-05-03)

## 1) 우선순위 높은 개선 (다음 4~6주)

1. **데이터 표준화 레이어 도입**
   - 앱(01/05/06/10)별로 저장 키를 공통 스키마(`lot_id`, `batch_no`, `eval_type`, `updated_at`, `owner`)로 정규화.
   - 효과: 대시보드 집계/검색 로직 단순화, 누락 데이터 탐지 자동화.

2. **통합 검색(Global Search) 추가**
   - Lot 번호/Batch 번호/고객사 코드 단일 검색창 제공.
   - 검색 결과에서 05(계보), 06(일정), 10(KPI)로 딥링크 이동.
   - 효과: "같은 Lot를 여러 탭에서 다시 찾는" 반복 작업 제거.

3. **이상치 알림 워크플로우**
   - 05 SQC 이탈 또는 10 KPI 임계치 초과 시 담당자/부서 단위 알림.
   - 상태값: `new → acknowledged → actioned → closed`.
   - 효과: 이슈 처리 리드타임 단축, 누가/언제 대응했는지 감사 추적 가능.

4. **권한 모델(RBAC) 최소 버전 적용**
   - 역할 예시: 관리자, 품질평가, 제조, 조회전용.
   - COA/사전심사/원본 업로드 등 민감 기능 권한 분리.
   - 효과: 운영 리스크 감소, 외부 감사 대응력 향상.

## 2) 중기 개선 (분기 단위)

1. **데이터 품질 모니터링 지표**
   - Completeness(필수값 채움률), Timeliness(지연시간), Consistency(시스템 간 일치율) KPI를 10번 대시보드에 추가.

2. **Lot Lifecycle 타임라인 화면**
   - "입고 → 합성 → 정제 → 평가 → COA" 이벤트를 시간축으로 시각화.
   - 병목구간(대기시간) 자동 하이라이트.

3. **변경 이력(Audit Log) 중앙화**
   - 주요 변경(삭제/수정/승인) 로그를 공통 포맷으로 저장.
   - 월간 리뷰 리포트 자동 생성 기반 마련.

## 3) 제안서 개선 포인트 (발표 관점)

1. **Slide 3의 ‘60% 완료’ 근거 수치화**
   - 예: 일평균 사용건수, 자동화 처리량, 수작업 절감시간(주/월).

2. **Slide 6~7 사례에 전/후 비교 추가**
   - Before: 담당자 추적시간, After: 처리시간 감소율.

3. **Slide 12 기대효과를 3개 숫자로 고정**
   - 예시: 리드타임 -30%, 재작업 -20%, 보고 준비시간 -40%.

## 4) 실행 권고

- **1단계(이번 달)**: 데이터 표준화 + 통합 검색 PoC.
- **2단계(다음 달)**: 이상치 알림/상태관리 + 기본 RBAC.
- **3단계(분기)**: 데이터 품질 KPI + Lifecycle 타임라인.
